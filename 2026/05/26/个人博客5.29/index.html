5.29 利用github和Hexo网站搭建个人博客

一、整体理解

  1. 免费托管需求可通过 GitHub 平台实现，Markdown 内容快速生成静态博客的需求可通过 Hexo 框架满足，两者结合即可打造一套零成本、易维护的个人博客系统。

  2. 你写文章（Markdown格式）
    ↓
    Hexo 生成网页
    ↓
    上传到 GitHub
    ↓
    GitHub Pages 自动变成网站

  3. 写博客，其实就是：写 md 文件 → 一键发布

二、准备环境

1.安装Node.js

如果不确定是否已经安装过，先打开cmd，在终端里输入

node -v

npm -v

如果出现版本号说明安装过

没有安装过的：

下载方法链接【2026 最新版】Node.js安装及环境配置超详细教程（以win11为例子）_nodejs安装及环境配置win11-CSDN博客

！！注意：这里还需要再加一个环境变量

step1：点击环境变量中的path

step2：点击新建，输入你安装的nodejs所在路径（我的在D:\node）

2.安装 Git

如果不确定是否已经安装过，先打开cmd，在终端里输入 git --version

如果出现版本号说明安装过

没有安装过的：

安装方法链接：【2026 最新版】Git安装及环境配置超详细教程（以win11为例子）_git安装及配置教程-CSDN博客

三、安装 Hexo

打开cmd或者powershell,输入: npm install -g hexo-cli

关闭终端然后重新打开一个

然后检查是否安装成功：hexo -v

安装成功会显示版本

四、创建Hexo项目

（接下来的演示我将博客放到D盘，你放置的位置按照你的电脑实际情况来，只要是放在非C盘都可以）

step1：创建博客

cmd切换根目录到D盘：在命令行输入 d:

切换后输入：hexo init blog

step2: 进入目录，安装依赖

cd blog

npm install

检查是否安装成功：npm -v

step3:本地启动博客

hexo s

浏览器打开：http://localhost:4000

到目前为止，你已经拥有博客了。

如果出现下面这种报错：考虑node和hexo版本不配对，把报错发给AI分析

八、部署

打开终端，根目录D:\blog>

清理：hexo clean #清理旧缓存

生成网页： hexo g #generate 生成网页

上传部署：hexo d #deploy 上传到 GitHub Pages

第一次会让你登录 GitHub，选择你的登陆方式就可以了

如果选择token登录方式：

  1. 进入 https://github.com/settings/tokens，Generate new token 选择Generate new token (classic)
  1. 填写note(不能为空)，勾选repo
  1. generate token

  1. 复制token

    ！！！注意

    • token只会显示一次，无法再次看到具体内容
    • Token 属于敏感信息，本质上相当于“免密码登录凭证”。如果被他人获取，可能会导致账号、代码或数据泄露。因此不要随意公开 Token，一旦泄露，应立即删除旧 Token 并重新生成新的 Token。
  1. 把刚刚复制的token粘贴到登录界面，然后sign in

登陆成功或者之前登录过 GitHub，Windows 会自动缓存 Git 凭据，hexo d 不会再次要求登录,终端会显示：

就可以通过网址：https://你的用户名.github.io或者你的用户名.github.io，访问你的个人博客

到此就代表你的个人博客基础开发完成！！

十二、后续改进方向

目前博客已经能够完成文章编写、本地预览以及 GitHub Pages 在线部署等基础功能，后续还可以继续从以下几个方向进行优化与扩展：

  1. 引入图床与图片管理工具（如 PicGo + GitHub 图床），解决本地图片无法在线访问的问题；
  2. 增加评论系统、访问统计、文章搜索等功能，提高博客的交互性与实用性；
  3. 配置 GitHub Actions 自动部署，实现文章更新后一键自动发布；
  4. 持续优化博客主题页面，包括首页布局、导航栏、背景图片与个人主页信息等；
  5. 后续可将博客作为个人学习记录平台，用于整理 Java、Python、算法、项目开发等技术内容，逐步形成个人技术知识库。
  6. ……

十三、图床

图床： 专门放图片的网络仓库

目前markdown里的图片路径是在本地电脑，而GitHub Pages 是公网网站，它根本没有你的 D 盘。访问你的个人博文，是无法看到markdown文件里的本路路径的图片，所有我们利用PicGo + GitHub 图床来让博客网页端也能看到图片

step1: 创建图床仓库

在github里面新建仓库，命名随意，权限公开

step2:安装PicGo

https://github.com/Molunerfinn/PicGo/releases

记住安装路径，后续会用到

step3：配置 PicGo（核心）

3.1 打开picgo，点击Picbeds Settings->Github->+

3.2 填写信息

Configuration Name：随便写一个名字，用来区分配置，比如myblog

Set Repo Name（最重要）: 填写GitHub用户名/仓库名，比如我的是：miao-22a/blog-images

Set Branch：一般是main(根据仓库目前分支填写，一般默认是main)

Set Token：和第八大点里一样，获取token(详细步骤在八、部署)

Set Path：建议填img/，这样图片会自动放进：仓库/img/，更整洁。也可以不填。

Set Custom URL（先别填）：先空着，后面如果你想加 CDN 再说。

最后点击confirm,创建成功如下所示：

step4：手动配置连接

4.1 在picgo的upload area界面上传图片

4.2 上传成功会：

  • Gallery会显示上传的图片

  • github仓库会显示

4.3 在picgo复制图片路径 此刻会显示路径在网页上

比如我这张会显示：https://raw.githubusercontent.com/miao-22a/blog-images/main/img/微信图片_20260224003141_42_19.jpg

此刻路径在公开的仓库里面，当你把这个图片嵌入到markdown文档里面并发布在个人博客，博客就可以通过公开的仓库网站找到图片从而显示。

step5：重新部署博客

大家可以新建一个图床测试的markdown文件，把图片传入picgo得到链接，再把链接复制到文件里面，然后跟着步骤十一、更新博客写一篇新博客看图床功能是否成功
